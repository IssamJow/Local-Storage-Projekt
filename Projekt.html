HTML LocalStorage Tutorial

Mit diesem Tutorial soll gezeigt werden, wie Local Storage überhaupt funktioniert und angewendet wird.
Außerdem werden die Arten des Local Storage voneinander unterschieden und in praktischen Beispielen demonstiert.
Zu guter Letzt werden die Vorteile gegeneüber Cookies dargestellt und gezeigt, wieso Local Storage eine bessere Anwendung für Webstorage ist.

Local Storage Definition

Mit Local Storage können Webanwendungen lokal Daten im Browser des Benutzers speichern.

Vor HTML5 mussten Anwendungsdaten in Cookies gespeichert werden, die in jeder Serverabfrage drin waren. Lokaler Speicher ist sicherer und große Datenmengen können lokal gespeichert werden, ohne die Performence der Webseite zu beeinflussen.

Im Gegensatz zu Cookies ist die Speichergrenze viel größer (mind.5MB) und Daten werden niemals an den Server übertragen.

Local Storage Arten

HTML Local Storage bietet 2 Arten für die Speicherung von Daten auf dem Client an:

  • window.localStorage - speichert Daten ohne Verfallsdatum
  • window.sessionStorage - speichert Daten für eine Sitzung (Daten gehen verloren, wenn die Registerkarte im Browser geschlossen wird)

Bevor man Local Storage anwenden kann, sollte man prüfen, ob der Browser localstorage-fähig ist:

if(typeof(Storage) !== "undefined") {
// Code for localStorage/sessionStorage.
} else {
// Sorry! No Web Storage support..
}

Das localStorage

Speichert die Daten ohne Ablaufsdatum. Die Daten werden nach dem Schließen des Browsers nicht gelöscht, sondern sind für den nächsten Tag , Woche oder Jahr wieder verfügbar.

Beispiel für localStorage:

// Store
localStorage.setItem( "studiengang" , "MedienInformatik");
// Retrieve
document.getElementById("result").innerHTML = localStorage.getItem( "studiengang");

Beispiel erklärt :

  • Erstellt ein name/value Paar mit dem Name="studiengang" und dem Wert="Medieninformatik".
  • ruft den Wert von "studiengang" auf, und fügt es in den Element mit dem id="result".
Das obere Beispiel könnte auch so aussehen:
// Store
localStorage.studiengang = "MedienInformatik;
// Retrieve
document.getElementById("result").innerHTML = localStorage.studiengang;

Die Syntax für das Entfernen von "studiengang" sieht folgendermaßen aus:

// Loescht "studiengang" Item.
localStorage.removeItem("studiengang");

Das Entfernen aller Werte des localStorage sieht so aus:

// Loescht alle Items des localStorage.
localStorage.clear();

Das sessionStorage

Das sessionStorage ist genauso wie localStorage, mit Ausnahme, dass die Daten nur für eine Sitzung gespeichert werden. Die Daten werden gelöscht, wenn der Benutzer das jeweilige Fenster/Tab schließt.

Beispiel für sessionStorage:

// Store
sessionStorage.setItem( "studiengang" , "MedienInformatik" );
// Retrieve
document.getElementById("result").innerHTML = sessionStorage.getItem( "studiengang");

Das folgende Besipiel zählt die Haüfigkeit der Klicks auf einen Button, in der aktuellen Sitzung:

Beispiel

if (sessionStorage.clickcount) {
sessionStorage.clickcount = Number(sessionStorage.clickcount) + 1;
} else {
sessionStorage.clickcount = 1;
}

sessionStorage.clickcount +" time(s) in this Session.";

LocalStorage vs SessionStorage

Die folgende Tabelle zeigt die Unterschiede zwischen den beiden Arten des Local Storages:

Storage-Arten Max. Größe Haltbarkeit Verfügbarkeit bei Fensterns/tabs Unterstützte Datentypen
LocalStorage 5MB pro App/ Browser(kann vom Benutzer erhöht werden ,falls nötig). Nicht von allen Browsern unterstützt. Gespeichert bis es vom Benutzer gelöscht wird (Cache löschen). Verteilt auf allen Fenstern/Tabs eines einzelnen Browsers. Nur String als "Key-value"-Paar.
SessionStorage Begrenzt auf Systemspeicher. Hält an bis das zugehörige Fenster/Tab geschlossen wird. Verfügbar nur innerhalb des Fensters/Tabs, von dem sie erstellt wurde. Nur String als "Key-value"-Paar.

Warum nutzt man Local Storage?

Folgendes Problem wurde festgestellt :

Daten sollen client-seitig ohne Haltbarkeitsdatum gespeichert werden.
Dies können z.B. Daten sein, die den Nutzer ermöglichen, auch offline zu arbeiten. Dabei soll kein Datenverlust entstehen.

Lösung:

Als bewährte Möglichkeit dienten bisher Cookies als Speicherung von Daten, die jedoch jedesmal von einem Server aufgerufen werden mussten, und somit zum Offlinebetrieb sich nicht eigneten.
Seit HTML5 dient das localStorage-Objekt als native Speicherung direkt im Browser an. Dies bedeutet, das jeder Browser auf dem Rechner des Nutzers seinen eigenen localStorage hat.

Cookies Definition

Cookies sind Daten, die in kleinen Textdateien auf dem Computer gespeichert werden.

Sie erlauben einen Webserver auf dem PC des Anwenders Informationen zu hinterlegen. Da der Webserver nicht direkt auf die Datenträger des Anwenders Zugriff hat, muss er hierfür den Browser nutzen.

Cookies wurden erzeugt, um das Problem "wie erinnere ich mich an die Daten des Benutzers" zu lösen.

  • Wenn ein Benutzer ein Webseite besucht, wurde sein Name als Cookie gespeichert.
  • Wenn der Benutzer das nächste Mal diese Webseite besucht, "erinnert" der Cookie sich an seinen Namen.
//Cookies werden in Name-Wert Paar gespeichert:
username=John Doe

Note: Einzelne Cookies haben eine Max.Größe von ca. 4KB und können pro Domain nur 20 mal abgeschickt werden.

Javascript kann mit dem Befehl document.cookie Cookies erstellen, lesen & löschen.
Mit JavaScript kann ein Cookie wie folgt erstellt werden:
document.cookie="username=John Doe";
Das obere Beispiel könnte auch so aussehen:
document.cookie="username=John Doe; expires=Thu, 18 Dec 2013 12:00:00 UTC";
Hier wird der Zerfallsdatum des Cookies festgelegt. Ansonsten: Cookie wird nach Schließen des Browsers gelöscht.
Mit Javascript kann ein Cookie wie folgt gelesen werden:
var x = document.cookie;
Mit Javascript kann man ein Cookie genauso verändern wie man es erstellt hat:
document.cookie="username=John Smith; expires=Thu, 18 Dec 2013 12:00:00 UTC";
Und um ein Cookie zu löschen, kann man einfach ein Zerfallsdatum als Parameter eingeben:
document.cookie="username=; expires=Thu, 18 Dec 1993 12:00:00 UTC";
